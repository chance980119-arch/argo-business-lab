# Day 1: 첫 PR 체험 - 자기소개 프로필 만들기

## 오늘의 목표

1. **계정 생성** - 이메일 인증으로 로그인
2. **프로필 등록** - 팀원 소개 페이지에 나를 추가
3. **첫 PR** - 브랜치 → 커밋 → PR → 머지 → 배포 체험

---

## Part 1: 계정 생성 & 로그인 (5분)

### Step 1: 사이트 접속

브라우저에서 접속:
```
https://argo-business-lab.vercel.app
```

### Step 2: 로그인

1. 왼쪽 사이드바 하단의 **"로그인"** 버튼 클릭
2. 이메일 주소 입력 → **"인증 코드 받기"** 클릭
3. 이메일로 받은 6자리 코드 입력
4. 로그인 완료!

> 💡 이메일이 안 오면 스팸함을 확인하세요.

---

## Part 2: 프로필 등록 (5분)

### Step 1: 팀원 소개 페이지 이동

사이드바에서 **"팀원 소개"** 클릭 (또는 `/about` 접속)

### Step 2: 프로필 작성

1. **"내 프로필 등록"** 버튼 클릭
2. 정보 입력:
   - 이름 (표시될 이름)
   - 팀/역할
   - 자기소개 (한두 줄)
   - 스킬 태그 (쉼표로 구분)
   - GitHub 사용자명

3. **저장** 클릭

이제 팀원 소개 페이지에 내 프로필 카드가 보입니다!

---

## Part 3: 첫 PR 체험 (20분)

이제 진짜 개발자처럼 코드를 수정하고 PR을 만들어봅니다.

### Step 1: 프로젝트 클론

AI 코딩 도구(Claude, Cursor 등)에게 요청:

```
argo-business-lab 저장소를 클론해줘.
저장소 주소: https://github.com/chance980119-arch/argo-business-lab
클론 후 pnpm install도 실행해줘.
```

### Step 2: 새 브랜치 생성

```
새 브랜치를 만들어줘.
브랜치 이름: feature/intro-{내이름}
예시: feature/intro-amy
```

### Step 3: 자기소개 파일 추가

```
profiles 폴더에 내 자기소개 파일을 만들어줘.
파일명: {내이름}.md

내용:
# {이름}

## 소속
{팀/부서}

## 한줄 소개
{자기소개}

## AI와 함께 배우고 싶은 것
{배우고 싶은 내용}
```

### Step 4: 커밋 & 푸시

```
변경사항을 커밋하고 GitHub에 푸시해줘.
커밋 메시지: "docs: {내이름} 자기소개 추가"
```

### Step 5: PR 생성

```
GitHub에서 Pull Request를 만들어줘.
제목: [Day 1] {내이름} 자기소개 추가
설명: 첫 번째 PR입니다! 자기소개 파일을 추가했습니다.
```

또는 GitHub 웹사이트에서 직접:
1. https://github.com/chance980119-arch/argo-business-lab 접속
2. 상단의 노란 배너 "Compare & pull request" 클릭
3. 제목과 설명 작성 후 "Create pull request"

### Step 6: 머지 & 배포 확인

1. Slack 채널에 PR 링크 공유
2. 리뷰 승인 받기
3. **"Merge pull request"** 클릭
4. 1-2분 후 라이브 사이트에서 확인!

## 문제 해결

### "permission denied" 오류

GitHub collaborator로 초대되어 있는지 확인하세요.
관리자에게 초대를 요청하세요.

### 이메일 인증 코드가 안 와요

1. 스팸함 확인
2. 이메일 주소 오타 확인
3. 1분 후 다시 시도

### 충돌(conflict)이 발생했어요

AI에게 요청:
```
main 브랜치의 최신 변경사항을 가져와서
내 브랜치에 머지해줘. 충돌이 있으면 해결해줘.
```
